# Пример: Minimal (минимальный валидный план)

Этот пример демонстрирует абсолютный минимум, необходимый для создания валидного плана opskarta.

## Что демонстрирует

- **Только обязательные поля** — `version` и `nodes`
- **Один узел** — с единственным обязательным полем `title`
- **Отсутствие файла представлений** — views являются опциональными

## Файлы

| Файл | Описание |
|------|----------|
| `project.plan.yaml` | Минимальный валидный план с одним узлом |

## Структура плана

```yaml
version: 1

nodes:
  root:
    title: "Минимальный проект"
```

## Что опущено

В этом примере намеренно опущены все опциональные элементы:

- `meta` — метаданные плана (id, title)
- `statuses` — кастомные статусы
- `kind` — тип узла (по умолчанию task)
- `status` — статус узла
- `parent` — родительский узел
- `start`, `duration`, `after` — планирование
- `notes` — заметки
- Файл `*.views.yaml` — представления

## Использование

### Валидация

```bash
cd specs/v1
python tools/validate.py examples/minimal/project.plan.yaml
```

Этот пример полезен как отправная точка для понимания формата. Для реальных проектов рекомендуется использовать более полную структуру — см. пример [hello](../hello/).
