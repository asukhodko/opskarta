# Contributing to opskarta

Спасибо, что хочешь помочь ❤️

## Основной принцип проекта

**Спецификация — ядро. Инструменты — плагины.**

Мы принимаем изменения, которые:
- улучшают ясность и совместимость формата;
- добавляют небольшие, хорошо изолированные “референсные” инструменты;
- не превращают проект в монолитную PM‑систему.

## Как предложить изменение

1. Создай issue: проблема/предложение/вопрос.
2. Если это изменение формата — приложи пример YAML “до/после”.
3. Если это инструмент — приложи пример входа и ожидаемый выход.

## Требования к PR

- Маленькие PR лучше больших.
- Новые поля в формате:
  - должны быть опциональными (если это не новая major‑версия),
  - должны иметь понятную семантику,
  - должны быть описаны в спецификации (см. `specs/v1/spec/`).
- Код:
  - Python >= 3.11
  - форматирование/линт: ruff (по возможности)
  - тесты: pytest (если логика нетривиальная)
